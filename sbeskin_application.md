# Бескин Семён - "FeedBackPic"


### Группа: 10 - И - 7
НАЗВАНИЕ ПРОЕКТА

«Семантический анализатор отзывов FeedBackPic»

ПРОБЛЕМНОЕ ПОЛЕ

Растёт объём текстовых отзывов. Бизнесу и исследователям нужен быстрый анализ тональности (позитив/негатив/нейтраль) без риска утечки данных. Существующие решения либо

требуют загрузки в облако (нарушая конфиденциальность), либо сложны и дороги (MAXQDA, NVivo), либо не понимают смысл текста (стандартный поиск ОС). Наш продукт работает

локально, прост в использовании и даёт наглядную визуализацию.

ЗАКАЗЧИК / ПОТЕНЦИАЛЬНАЯ АУДИТОРИЯ

Продукт полезен разным группам:

Владельцы малого бизнеса (анализ отзывов с карт и маркетплейсов)

Маркетологи (выявление негативных тем)

Студенты и исследователи (обработка опросов)

Преподаватели (демонстрация NLP на уроках)

Некоммерческие организации (анализ обращений без передачи данных в облако)

АППАРАТНЫЕ / ПРОГРАММНЫЕ ТРЕБОВАНИЯ

Десктопное приложение. Версии:

Windows 10 и выше: ~4 ГБ ОЗУ, ~500 МБ диска

ФУНКЦИОНАЛЬНЫЕ ТРЕБОВАНИЯ

Загрузка файлов и папок (.txt, .docx, .pdf, .xlsx, .csv)

Определение языка (русский, английский)

Классификация тональности с оценкой уверенности

Таблица с фильтрацией и сортировкой

Визуализация: круговые и столбчатые диаграммы, облако слов

Экспорт в PDF, Excel, CSV

Сохранение/загрузка результатов в JSON

Индикатор прогресса при обработке

ПОХОЖИЕ / АНАЛОГИЧНЫЕ ПРОДУКТЫ

Google Cloud NLP — требует интернета и загрузки данных на сервер, платный.

MAXQDA — дорогой ($500+), сложный интерфейс.

Библиотеки NLTK/spaCy — нет GUI, нужны навыки программирования.

ИНСТРУМЕНТЫ РАЗРАБОТКИ, ИНФОРМАЦИЯ О БД

Язык: Python 3.10+

GUI: Tkinter / PyQt

Тональность: transformers (rubert-tiny-sentiment-balanced)

Текст: nltk, spacy, pymorphy2

Чтение файлов: python-docx, pdfplumber, openpyxl, pandas

Визуализация: matplotlib, wordcloud

Сборка: PyInstaller

БД: SQLite / JSON

ЭТАПЫ РАЗРАБОТКИ

Анализ требований и сценариев

Модуль извлечения текста из файлов

Интеграция модели тональности

Разработка графического интерфейса

Визуализация (диаграммы, облако слов)

Экспорт в PDF, Excel, CSV

Тестирование на реальных отзывах

Сборка дистрибутива

Документация и подготовка к защите

ВОЗМОЖНЫЕ РИСКИ

Неточная тональность — используем несколько моделей, добавим ручную правку.

Проблемы с PDF — ограничим поддерживаемые PDF, применим несколько библиотек.

Нехватка времени на изучение технологий — выберем простую модель на старте.

Неудобный интерфейс — проведём раннее тестирование на знакомых.

Загруженность учёбой — сделаем сначала минимальную версию, остальное потом.
